# 决策层 (Center Business)

[English](centerBusiness.en.md) | 中文

## 1. 概述

决策层 (`centerBusiness`) 是 `arnics-os` 仿生社会架构中的“决策执行部门”。它专门负责处理与具体产品相关的高层决策与业务逻辑。通过将决策逻辑与底层驱动、系统事件分发相隔离，决策层确保了代码的高内聚与低耦合。

## 2. 职责定位

在“企业社会”模型中，决策层类似于公司的核心决策团队：
- **不关心底层细节**：不直接操作硬件外设，所有硬件交互通过 `DataPlat` 或 `Drivers` 完成。
- **业务编排**：接收来自 `centerEvent` (事件中心) 的任务分配，编排具体的决策流与业务流程。
- **状态维护**：根据决策流转，更新系统状态，并下发指令给 `centerMedia` (媒体中心) 进行 UI 或音频反馈。

## 3. 核心文件与接口

- `business_process.h/c`：决策层的总调度与流转逻辑入口。
- `entry_business_api.h/c`：决策层对外提供的 API 接口（对其他部门暴露的职能）。

### 主要接口

```c
// 决策层主处理流程（通常在 RTOS 任务中循环调用或由定时器驱动）
extern void business_process(void);

// 决策层初始化
extern void business_init(void);
```

## 4. 最佳实践

在开发新的业务与决策功能时：
1. **模块化设计**：将复杂的逻辑拆分为多个子模块，在 `business_process` 中进行统一调度。
2. **事件驱动**：决策模块不应使用死循环等待（除非在独立的 RTOS 任务中），而应该基于状态机或事件驱动模型。
3. **数据隔离**：决策过程中产生的持久化数据或配置，应交由 `DataPlat` 进行管理。
