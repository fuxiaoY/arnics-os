# 督察中心 (Center Guard)

[English](centerGuard.en.md) | 中文

## 1. 概述

督察中心 (`centerGuard`) 是 `arnics-os` 架构中的“安全保卫部门”。它的核心职责是监控系统的健康状态，确保系统在异常情况下能够安全恢复，类似于企业的保安或合规审查部门。

## 2. 职责定位

- **看门狗管理 (Watchdog)**：负责软硬件看门狗的定期喂狗 (Feed) 动作。如果关键任务卡死，督察中心将停止喂狗，触发系统复位。
- **异常监控 (Exception Monitoring)**：监测各部门（如决策层、事件中心、媒体中心）的运行状态、内存使用率及 CPU 负载。
- **错误恢复 (Error Recovery)**：当检测到非致命错误时，尝试重启相关模块；遇到致命错误时，记录现场信息并安全重启。
- **日志与审计 (Logging & Audit)**：关键的安全事件和异常状态通过日志系统 (`uflog`) 记录，便于事后追溯。

## 3. 核心文件与接口

- `guard_process.h/c`：督察中心的监控与巡检逻辑入口。
- `entry_guard_api.h/c`：督察中心对外提供的注册与心跳接口。

### 主要接口

```c
// 督察中心主处理流程（定期巡检系统健康状态）
extern void guard_process(void);
```

## 4. 最佳实践

1. **心跳注册**：建议各关键 RTOS 任务向督察中心注册心跳（Heartbeat）。
2. **分级处理**：针对不同级别的异常采取不同的恢复策略（Warning -> 局部重置 -> 全局重启）。
3. **硬件隔离**：看门狗的具体操作应通过 `drivers/interface_iwdg` 抽象层完成，保持平台无关性。
